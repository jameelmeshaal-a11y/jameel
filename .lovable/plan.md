

# ضمان عدم تكرار مشكلة التسعير الخاطئ — 3 إجراءات وقائية

## المشكلة
المحرك يختار أعلى درجة تشابه بدون فحص منطقية السعر. بند رشاشات (312 ريال) حصل على سعر نظام غاز NOVEC (78,000 ريال) لأن كلاهما في فئة Fire Fighting ونفس الوحدة.

## الحل — 3 طبقات حماية

### 1. حارس ضخامة السعر (Price Magnitude Guard) — `matchingV3.ts`

بعد اختيار أفضل مطابقة (سطر 154-158)، نجمع كل المطابقات التي حصلت على درجة ≥50 ونقارن أسعارها. إذا كان الفرق بين أعلى وأقل سعر أكثر من **50 ضعف**:
- نختار البند **الأقرب نصياً** (أعلى text score) بدلاً من أعلى إجمالي
- نخفّض الثقة بـ 15 نقطة
- نضيف ملاحظة تحذيرية

```text
مثال:
  Match A: NOVEC → 78,000 ر.س (total: 60, text: 25)
  Match B: رشاش → 312 ر.س (total: 58, text: 40)
  Ratio: 250x → يتجاوز الحد (50x)
  → اختيار B (أعلى text score) بثقة مخفّضة
```

### 2. توسيع Anti-Confusion لأنظمة الحريق — `synonyms.ts`

إضافة مفاهيم جديدة وأزواج متعارضة:

```typescript
// مفاهيم جديدة
"نظام_غاز_اطفاء": ["NOVEC", "FM200", "غاز صافي", "نظام إطفاء بالغاز", "gas suppression"],
"نظام_رغوي": ["إطفاء رغوي", "foam system", "رغوه"],

// أزواج Anti-Confusion الجديدة
["نظام_غاز_اطفاء", "رشاش_حريق"],
["نظام_غاز_اطفاء", "رشاش_جانبي"],
["نظام_غاز_اطفاء", "رشاش_متدلي"],
["نظام_غاز_اطفاء", "رشاش_قائم"],
["نظام_رغوي", "رشاش_حريق"],
["نظام_رغوي", "نظام_غاز_اطفاء"],
```

### 3. إصلاح البند المتضرر — Database

إزالة الربط الخاطئ من البند الحالي وإعادة حالته لـ `pending`.

## الملفات المتأثرة

| الملف | التغيير |
|---|---|
| `src/lib/pricing/matchingV3.ts` | إضافة Price Magnitude Guard بعد حلقة التقييم |
| `src/lib/pricing/synonyms.ts` | إضافة مفاهيم أنظمة الإطفاء + أزواج Anti-Confusion |
| Database | إصلاح البند المتضرر |

## ما لا يتأثر
- البنود المسعّرة سابقاً بشكل صحيح — لن تتغير
- خوارزمية النص الأساسية — بدون تعديل
- نظام الاعتماد والملاحظات — بدون تعديل
