

# فحص شامل واكتشاف المشاكل المتبقية — خطة معالجة

## نتائج الاختبارات

### المشاكل المكتشفة (31 بند مشبوه)

| المشكلة | العدد | مثال |
|---|---|---|
| **1. نظام التحكم في الأبواب مربوط بنظام CCTV** | 13 بند | "بوابة كشف المعادن" مسعّرة بـ 18,000 ← مربوطة بـ "شاشة مراقبة LCD" سعرها 812.5 (22x) |
| **2. بلاطات خرسانية مربوطة بكمرات نقل** | 6 بنود | "بلاطات خرسانية سمك 12/15/20 سم" ← كلها مربوطة بـ "كمرات نقل" بسعر 1,265 |
| **3. أنظمة اتصالات بفارق سعر كبير (5-11x)** | 5 بنود | لوحات توزيع مسعّرة بـ 140,000 vs مكتبة 13,000 |
| **4. مواسير صرف مربوطة ببند إنشائي** | 2 بند | مواسير HDPE مسعّرة بـ 750 vs مكتبة 130 |
| **5. عزل مجاري هواء مربوط بتشطيبات** | 2 بند | عزل HVAC duct مربوط ببند تشطيب خاطئ |

### السبب الجذري المشترك
المحرك يعتمد على **تشابه النص العام** بدون فهم **النوع الهيكلي** للبند. مثلاً:
- "بلاطات خرسانية" و"كمرات نقل" كلاهما يحتوي "خرسانة مسلحة" — فيتم ربطهما
- "نظام التحكم في الأبواب" و"نظام CCTV" كلاهما يحتوي "أنظمة أمنية" — فيتم ربطهما
- لا يوجد Anti-Confusion بين هذه الفئات المختلفة

## الحل — 4 إجراءات

### 1. توسيع المرادفات وAnti-Confusion — `synonyms.ts`

**مرادفات جديدة:**
```text
"بلاطات_خرسانية": ["بلاطات خرسانيه", "بلاطه خرسانيه", "concrete slab", "slab", "بلاطه ارضيه"]
"كمرات_خرسانية": ["كمرات", "كمره", "beam", "كمرات نقل", "كمرات معلقه", "كمرات ارضيه"]
"نظام_تحكم_ابواب": ["تحكم في الابواب", "access control", "تحكم ابواب", "door control"]
"نظام_CCTV": ["دائره تلفزيونيه مغلقه", "CCTV", "كاميرات مراقبه", "مراقبه تلفزيونيه"]
"بوابة_كشف_معادن": ["بوابه كشف معادن", "metal detector", "كشف معادن"]
"مواسير_صرف": ["مواسير صرف", "drainage pipe", "صرف صحي", "مواسير HDPE"]
"عزل_مجاري_هواء": ["عزل مجاري هواء", "duct insulation", "عزل دكت"]
```

**أزواج Anti-Confusion:**
```text
["بلاطات_خرسانية", "كمرات_خرسانية"]  — منع خلط بلاطة مع كمرة
["نظام_تحكم_ابواب", "نظام_CCTV"]      — منع خلط أنظمة مختلفة
["نظام_تحكم_ابواب", "كاميرا_مراقبة"]   — منع خلط
["بوابة_كشف_معادن", "نظام_CCTV"]      — منع خلط
["بوابة_كشف_معادن", "كاميرا_مراقبة"]   — منع خلط
["مواسير_صرف", "خرسانة"]              — منع خلط مواسير بإنشائي
["عزل_مجاري_هواء", "عزل_مائي"]        — منع خلط
["عزل_مجاري_هواء", "لياسه"]           — منع خلط
```

### 2. تشديد Price Magnitude Guard — `matchingV3.ts`
حالياً العتبة 50x (مرتفعة جداً). تخفيضها إلى **10x** لالتقاط حالات مثل 22x و15x التي مرّت بدون اعتراض.

### 3. إعادة تعيين 31 بند متضرر — Database
إعادة جميع البنود المشبوهة (ratio > 5x أو مربوطة بفئة خاطئة) إلى `pending`.

### 4. إضافة Structural Type Gate — `matchingV3.ts`
حاجز جديد: إذا كان البند يذكر نوعاً هيكلياً محدداً (بلاطة، كمرة، عمود، قاعدة) والمرشح يذكر نوعاً مختلفاً ← تخطي كامل. هذا أقوى من Anti-Confusion لأنه يعمل على مستوى الكلمات المباشرة بدون الحاجة لتعريف كل زوج.

## الملفات المتأثرة

| الملف | التغيير |
|---|---|
| `src/lib/pricing/synonyms.ts` | إضافة 7 مجموعات مرادفات + 8 أزواج Anti-Confusion |
| `src/lib/pricing/matchingV3.ts` | تخفيض عتبة Magnitude Guard من 50x إلى 10x + إضافة Structural Type Gate |
| Database | إعادة تعيين ~31 بند متضرر لـ pending |

## النتيجة المتوقعة
- بلاطات لن تُربط بكمرات
- أنظمة أمنية مختلفة لن تتداخل
- أي فارق سعر > 10x سيُفحص تلقائياً
- البنود المتضررة ستُعاد لـ pending للتسعير الصحيح
