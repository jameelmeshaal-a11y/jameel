

# تعديل الوحدة في جدول الكميات مع الانعكاس على مكتبة الأسعار

## الملخص

إضافة إمكانية تعديل عمود "الوحدة" مباشرة في جدول الكميات (BoQTable)، بحيث يتم تحديث **البيانات** في قاعدة البيانات: جدول `boq_items` + جدول `rate_library` المرتبط.

## ما سيحدث في قاعدة البيانات

| الجدول | نوع التغيير | التفصيل |
|---|---|---|
| `boq_items` | تحديث بيانات ✅ | تعديل قيمة عمود `unit` للبند المعدّل |
| `rate_library` | تحديث بيانات ✅ | تعديل قيمة عمود `unit` للبند المرتبط عبر `linked_rate_id` |
| هيكل الجداول | بدون تغيير ❌ | لا أعمدة جديدة، لا migrations |

## التغييرات في الكود

### `src/components/BoQTable.tsx`

1. إضافة state لتتبع تعديل الوحدة (`editingUnitId`, `editingUnitValue`)
2. النقر المزدوج على خلية الوحدة → يظهر حقل إدخال
3. عند الحفظ (Enter أو blur):
   - تحديث `boq_items.unit` في قاعدة البيانات
   - إذا كان البند مرتبطاً (`linked_rate_id`) → تحديث `rate_library.unit` أيضاً
   - رسالة نجاح توضح أن التعديل انعكس على المكتبة
4. Escape → إلغاء التعديل
5. أيقونة قلم تظهر عند تمرير الماوس
