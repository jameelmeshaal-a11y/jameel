| المشكلة | مثال فعلي | السبب |
|---|---|---|
| **بنود مختصرة بدون سياق** | `قطر 20 مم` مطابق لـ "أنابيب صرف UPVC 20مم" بثقة 53% فقط | البند يعتمد على عنوان الصف الأعلى (مواسير صرف) لكن السياق لم يُدمج بشكل كافٍ |
| **أبعاد مختلفة = نفس البند** | `قطر32 مم` مطابق لـ "أنابيب قطر 20 مم" | النظام يطابق نصياً بدون فهم أن 32≠20 |
| **نموذج مختلف = نفس البند** | `MD-14` مطابق لـ `MD-05` (باب مختلف الأبعاد) بثقة 72% | النظام يعطي +40 لأي code match لكن لا يتحقق أن الكود نفسه |
| **صمام بوابة ≠ منفس هواء** | `صمام بوابة قطر 25مم` مطابق لـ `منفس هواء AAV قطر 25مم` بثقة 50% | التشابه في "قطر 25 مم" يرفع النتيجة رغم اختلاف البند كلياً |
| **مرادفات غير مدعومة** | "جانبي" ≠ "متدلي" في رشاشات الحريق | لا توجد طبقة مرادفات |

### التحسينات التقنية

#### 1. استخراج الأبعاد والأقطار (Dimension Parsing)
```text
"قطر 20 مم" → { type: "diameter", value: 20, unit: "mm" }
"1200*2000مم" → { type: "dimensions", values: [1200, 2000] }
"MD-14" → { type: "model", code: "md-14" }

القاعدة: إذا البند الأول فيه diameter=32 والمكتبة فيها diameter=20
→ خصم -25 نقطة (بند مختلف المقاس)
→ لكن إذا لم يوجد بند بنفس القطر → يبقى كأفضل تطابق مع تحذير
```

#### 2. فهم السياق الرأسي (Parent Context Boost)
```text
الحالي: "قطر 20 مم" → يبحث بهذا النص فقط
المحسّن: "قطر 20 مم" + notes تحتوي [PARENT: أنابيب صرف صحي UPVC]
→ الوصف المدمج: "أنابيب صرف صحي UPVC — قطر 20 مم"
→ الثقة ترتفع من 53% إلى 85%+
```
السياق الرأسي موجود فعلاً في `mergedDescription` من `boqRowGrouping.ts` — المشكلة أن بعض البنود لا تمر عبر الدمج بشكل صحيح. سيتم تحسين استغلال `notes[PARENT:]` كمصدر ثانوي.

#### 3. طبقة مرادفات قابلة للتوسعة
```text
synonyms.ts:
{
  "صمام_بوابة": ["صمام بوابة", "gate valve", "محبس"],
  "منفس_هواء": ["منفس هواء", "AAV", "air admittance"],
  "رشاش_حريق": ["رشاش حريق", "sprinkler", "رشاش"],
  "متدلي": ["متدلي", "pendent", "معلق"],
  "جانبي": ["جانبي", "sidewall", "حائطي"],
  ...
}
→ قبل المطابقة: توسيع الوصف بالمرادفات المرتبطة
→ منع الخلط بين فئات مختلفة (صمام ≠ منفس)
```

#### 4. نظام Scoring V3 — ترجيح محسّن
```text
الحالي:
  textSimilarity × 60 + category × 15 + keywords × 25 + codes × 40 + containment × 20

المحسّن:
  textSimilarity × 50         (أساسي — لكن أقل وزناً لصالح الدقة)
  + categoryMatch × 15        (بدون تغيير)
  + keywordOverlap × 20       (بدون تغيير)
  + exactCodeMatch × 40       (فقط إذا الكود متطابق تماماً — ليس "أي كود")
  + codeMismatchPenalty × -25  (جديد: خصم إذا الكودين مختلفين)
  + dimensionMatch × +15      (جديد: مكافأة إذا القطر/المقاس متطابق)
  + dimensionMismatch × -20   (جديد: خصم إذا القطر/المقاس مختلف)
  + synonymBoost × +10        (جديد: مكافأة من المرادفات)
  + parentContextBoost × +15  (جديد: سياق من الصف الأعلى)
  + antiConfusionGate          (جديد: منع خلط فئات غير متوافقة)
```

#### 5. Anti-Confusion Gate — منع المطابقات الخاطئة
```text
أزواج ممنوعة من التطابق:
  صمام_بوابة ↔ منفس_هواء
  كابل_كهرباء ↔ أنابيب_صرف
  رشاش_جانبي ↔ رشاش_متدلي (نفس الفئة لكن نوع مختلف)

إذا البند من فئة A والمرشح من فئة B وهما ممنوعان:
→ تصفير النتيجة بالكامل
```

#### 6. Feature Flag للتبديل الآمن
```typescript
// في pricingEngine.ts
const USE_MATCHING_V3 = true; // يمكن إيقافه فوراً للرجوع

function findRateLibraryMatch(...) {
  if (USE_MATCHING_V3) {
    return findRateLibraryMatchV3(...); // من matchingV3.ts
  }
  return findRateLibraryMatchLegacy(...); // الكود الحالي بدون تغيير
}
```

### الحوكمة والحماية

| الإجراء | التفاصيل |
|---|---|
| **Feature Flag** | `USE_MATCHING_V3` — يمكن إيقافه فوراً للرجوع للنظام القديم |
| **عزل الكود** | ملفات جديدة فقط (`matchingV3.ts`, `synonyms.ts`) — الملفات الحالية لا تُمس |
| **Contract Stability** | نفس توقيع `findRateLibraryMatch` — نفس المدخلات والمخرجات |
| **Regression Tests** | اختبارات تتحقق أن البنود الحالية لا تتأثر سلباً |
| **Logging** | كل قرار مطابقة يُسجَّل مع التفسير في `notes` |
| **Audit Trail** | `confidence` + `source` + `notes` تبقى كما هي |
| **Backward Compatible** | أي بند كان يعمل قبل التحسين سيعمل بعده أو أفضل |

## معايير القبول

1. البنود المختصرة مثل "قطر 20 مم" مع سياق → ثقة ≥75%
2. مطابقات خاطئة (صمام↔منفس) → تُمنع تلقائياً
3. أبعاد/أكواد مختلفة → خصم واضح في النتيجة
4. جميع الـ 730 بند approved حالياً → تبقى approved أو ترتفع
5. اختبارات regression تنجح 100%
6. Feature flag يعمل بشكل سليم للرجوع السريع
